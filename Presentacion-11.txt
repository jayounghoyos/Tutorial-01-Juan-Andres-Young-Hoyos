Este tutorial muestra cómo desplegar los proyectos “frontend” y “backend” usando Docker
y Docker-compose.

A.Creación de la máquina virtual
Crear la máquina virtual para desplegar el proyecto
•
•

Siga los pasos A, B, C y D del “Tutorial-GCP-Completo” para crear una nueva máquina virtual.
Una vez creada esa máquina, ya tendrá instalado Docker y Docker Compose.
Ahora, guarde la dirección IP que le arroja esa máquina (ver siguiente imagen).

Crear el Dockerfile
•

Ubicado en la carpeta raíz del proyecto (“backend”), cree el archivo Dockerfile con el
siguiente contenido:
Añadir todo este código

FROM node:22-bookworm-slim
WORKDIR /app
ENV NODE_ENV=production
COPY package*.json ./
RUN npm install --omit=dev
COPY dist ./dist
EXPOSE 3000
CMD ["node", "dist/main.js"]

Compilar el proyecto local
•

Ejecute el siguiente comando ubicado en la carpeta raíz de su proyecto “backend”:

Ejecutar en Terminal

D. Despliegue
Desplegar el proyecto
•
•

Acceda por SSH a la máquina virtual.
Cree una regla de firewall que permita acceder al puerto 3000. Pregúntele a ChatGPT como
hacerlo, o tome de base el siguiente pantallazo.

•
•
•

Clone su repositorio GitHub.
Ubíquese dentro de la carpeta principal de su repositorio.
Ejecute el siguiente comando ubicado en la carpeta que contiene el docker-compose.yml:
Ejecutar en Terminal

docker compose up -d

•

Si todo sale bien, deberá ver desplegada en la dirección IP de su proyecto el proyecto
FullStack (ver siguientes figuras).
